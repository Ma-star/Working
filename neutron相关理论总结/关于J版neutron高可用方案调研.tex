		\item[3.]尽力考虑负载均衡
		\item[4.]尽可能简单，不要太复杂。
	\end{itemize}

\section{使用DVR方案}
\subsection{DVR优势}
\begin{itemize}
	\item[1.]未使用DVR的neutron，内部不同子网之间的vm信息交互，以及vm访问外网，都需要走网络节点的路由。增加了网络的负载，同时网络节点非HA。
	\item[2.]DVR可以将不同子网vm的信息交互，走计算节点的路由，分担了部分网络节点的负载。
\end{itemize}
\subsection{DVR存在的问题}
\begin{itemize}
	\item[1.]未分配浮动ip的vm，访问外网，仍旧要走网络节点。
	\item[2.]在DVR的同时，网络节点也要做HA(例如使用pacemaker)。遗憾的是，J版neutron不同时支持HA和DVR。
\end{itemize}
\subsection{DVR后续版本的开发}
\begin{itemize}
	\item[1.]kilo版本支持HA+DVR，同时支持内网Vlan（J版DVR只支持vxlan或者gre）
	\item[2.]liberty版本支持分布式SNAT
\end{itemize}
